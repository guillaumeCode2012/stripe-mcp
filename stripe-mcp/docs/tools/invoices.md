# Invoices

Retrieve, list, pay, void, finalize, and send Stripe invoices. Invoices are
statements of what a customer owes — they're auto-generated by subscriptions
each billing cycle or created manually for one-time charges.

All tools validate input with zod and return the Stripe Invoice object as
pretty-printed JSON on success or a human-readable error string on failure.
`stripe_invoices_list` auto-paginates and wraps results as
`{ total_count, has_more, data }`.

---

### stripe_invoices_get

Retrieve an invoice by ID. Use when inspecting a single invoice's line items,
totals, or status, confirming an invoice was paid before granting access, or
showing a customer their invoice-history detail.

**Parameters**

| Param        | Type     | Required | Description                                            |
| ------------ | -------- | -------- | ------------------------------------------------------ |
| `invoice_id` | string   | yes      | ID of the invoice to retrieve (`in_...`)               |
| `expand`     | string[] | no       | Stripe expandable fields (e.g. `["customer", "subscription", "payment_intent", "lines.data.price"]`) |

**Returns** — the Stripe Invoice object.

**Stripe docs** — https://stripe.com/docs/api/invoices/retrieve

---

### stripe_invoices_list

List invoices with optional filters. Use when showing a customer their invoice
history, finding open or past-due invoices for collection workflows, or
auditing drafts before finalization.

**Parameters**

| Param            | Type    | Required | Description                                                                  |
| ---------------- | ------- | -------- | ---------------------------------------------------------------------------- |
| `limit`          | integer | no       | Page size for each Stripe request (1-100)                                    |
| `customer`       | string  | no       | Only return invoices for this customer (`cus_...`)                           |
| `status`         | string  | no       | `draft` \| `open` \| `paid` \| `uncollectible` \| `void`                     |
| `subscription`   | string  | no       | Only return invoices for this subscription (`sub_...`)                       |
| `starting_after` | string  | no       | Pagination cursor — ID of the last object from the previous page             |
| `max_items`      | integer | no       | Hard cap on total items returned. Defaults to 1000.                          |

**Returns** — `{ total_count, has_more, data: Invoice[] }`.

**Stripe docs** — https://stripe.com/docs/api/invoices/list

---

### stripe_invoices_pay

Pay an invoice (draft or open) immediately. Use when a customer clicks "Pay
now" on an open invoice, you need to advance an invoice to paid status outside
the normal billing cycle, or you're marking an invoice as paid out-of-band
(cash, check, wire transfer).

**Parameters**

| Param              | Type     | Required | Description                                                                  |
| ------------------ | -------- | -------- | ---------------------------------------------------------------------------- |
| `invoice_id`       | string   | yes      | ID of the invoice to pay (`in_...`). Must be in draft or open status.        |
| `paid_out_of_band` | boolean  | no       | If `true`, mark as paid without charging through Stripe (cash, wire, etc.)   |
| `source`           | string   | no       | ID of a payment source (`card_...`, `src_...`) to charge                     |
| `payment_method`   | string   | no       | ID of a PaymentMethod (`pm_...`) to charge                                   |
| `mandate`          | string   | no       | ID of a Mandate to use for this payment                                      |
| `expand`           | string[] | no       | Stripe expandable fields (e.g. `["payment_intent"]`)                         |

**Returns** — the Stripe Invoice object after payment is attempted
(`status: 'paid'` or `'open'`).

**Stripe docs** — https://stripe.com/docs/api/invoices/pay

### stripe_invoices_void

Mark an open invoice as void. Use when an invoice was issued in error and the
customer should never be charged, a disputed charge results in cancellation,
or you need to cleanly cancel an invoice without deleting it (for audit
trail). Voiding is irreversible.

**Parameters**

| Param        | Type     | Required | Description                                            |
| ------------ | -------- | -------- | ------------------------------------------------------ |
| `invoice_id` | string   | yes      | ID of the invoice to void (`in_...`). Must be in open status. |
| `expand`     | string[] | no       | Stripe expandable fields (e.g. `["customer"]`)         |

**Returns** — the Stripe Invoice object (`status: 'void'`).

**Stripe docs** — https://stripe.com/docs/api/invoices/void

---

### stripe_invoices_finalize

Finalize a draft invoice, advancing it to open status. Use when you want to
lock in line items and email the customer, trigger payment collection on a
manually-created draft, or convert a draft to an open invoice before charging.

**Parameters**

| Param         | Type     | Required | Description                                                                                |
| ------------- | -------- | -------- | ------------------------------------------------------------------------------------------ |
| `invoice_id`  | string   | yes      | ID of the draft invoice to finalize (`in_...`). Must be in draft status.                  |
| `auto_advance`| boolean  | no       | If `true` (default), Stripe auto-advances the invoice through the billing cycle.           |
| `expand`      | string[] | no       | Stripe expandable fields (e.g. `["customer", "payment_intent"]`)                           |

**Returns** — the Stripe Invoice object (`status: 'open'`).

**Stripe docs** — https://stripe.com/docs/api/invoices/finalize

---

### stripe_invoices_send

Email an invoice to the customer. Use when the invoice uses `send_invoice`
collection method, re-sending a reminder, or manually triggering invoice
delivery outside the auto-advance flow.

**Parameters**

| Param        | Type     | Required | Description                                                                              |
| ------------ | -------- | -------- | ---------------------------------------------------------------------------------------- |
| `invoice_id` | string   | yes      | ID of the invoice to email (`in_...`). Must be open with `collection_method=send_invoice`.|
| `expand`     | string[] | no       | Stripe expandable fields (e.g. `["customer"]`)                                           |

**Returns** — the Stripe Invoice object (sent to the customer email on file).

**Stripe docs** — https://stripe.com/docs/api/invoices/send
